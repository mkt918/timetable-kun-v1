# 時間割くんv2 テスト仕様書

> 手動テスト用チェックリスト。各項目をブラウザで実際に操作して確認する。

---

## 前提準備

| 項目 | 設定内容 |
|------|---------|
| 教員 | 田中（A）、鈴木（B）、山田（C）を登録 |
| 科目 | 数学、体育、英語を登録 |
| クラス | 1-1、1-2、1-3、2-1 が存在すること（config.js に定義済み） |
| 担当設定 | 1-1の数学=田中、1-2の数学=鈴木、1-1の体育=田中・鈴木（TT用） |
| クラス別設定 | 1-1の数学: 週4コマ、1-1の体育: TT・2コマ連続、1-1の英語: 合同（1-2と）|

## 1. クラス別設定（カリキュラム）

### 1-1. 科目追加・削除
- [ ] 「+科目追加」から数学を追加 → カードが表示される
- [ ] カード右上の削除ボタン → 確認後に削除される
- [ ] 週時間数をインライン編集（1→4に変更） → 即時保存される

### 1-2. 連続コマ設定
- [ ] セレクト「連続なし」→「2コマ連続」に変更 → カードにオレンジ「2連」バッジが出る
- [ ] 設定保存後にページリロード → 設定が保持されている

### 1-3. TT設定
- [ ] セレクト「通常」→「TT（複数教員）」に変更 → 紫「TT」バッジが出る
- [ ] 「通常」に戻す → バッジが消える

### 1-4. 合同設定 — ダイアログ
- [ ] 「合同（複数クラス）」に変更 → 自動で合同ダイアログが開く
- [ ] ダイアログに同学年クラスが一覧表示される
- [ ] **科目未設定** のクラス（相手に英語カリキュラムがない）→ チェックボックスが無効（クリックできない）かつ赤破線ボーダー
- [ ] 1-2にチェック → 「選択中: 1クラス（自クラス含め 2 / 6）」と表示
- [ ] 6クラス目のチェックを試みる → 5クラス目で上限に達しチェック不可になる
- [ ] 「保存してリンク」→ 1-1と1-2の双方に「合同リンク済」が表示される
- [ ] 1-2のカリキュラムを開く → 1-2の英語にも「合同」設定が自動で入っている（双方向リンク確認）
- [ ] 1-1の英語合同から1-2を外す → 1-2の英語も自動的に合同解除される

## 2. 手動配置（授業セル操作）

### 2-1. 通常配置
- [ ] 教員行の空きセルをクリック → モーダルが開く
- [ ] 科目・クラスを選択して配置 → 時間割に表示される
- [ ] トースト「授業を配置しました」が出る
- [ ] Ctrl+Z でUndoできる

### 2-2. TT自動配置
- [ ] 1-1の体育（TT設定・田中+鈴木担当）を田中の行から配置
- [ ] → 田中と鈴木が両方 teacherIds に入った状態で配置される
- [ ] トースト「授業を配置しました（TT）」と表示される

### 2-3. 連続コマ自動配置
- [ ] 1-1の体育（2コマ連続設定）を月曜2限に配置
- [ ] → 月曜2限・3限の両方に体育が入る
- [ ] トースト「授業を配置しました（2コマ連続）」と表示される
- [ ] 2限・3限のどちらかが既に埋まっている場合 → 確認ダイアログが出る
  - [ ] キャンセルで何も配置されない
  - [ ] OK で空いているコマのみ配置される
- [ ] Ctrl+Z で2コマ同時にUndoされる

### 2-4. 合同自動配置
- [ ] 1-1の英語（1-2と合同設定）を月曜1限に配置
- [ ] → 1-1・1-2の両クラスに同時に英語が入る
- [ ] トースト「授業を配置しました（2クラス合同）」と表示される
- [ ] 合同クラスの一方が既に埋まっている場合 → 確認ダイアログが出る

### 2-5. 連続コマ+合同 組み合わせ
- [ ] 1-1の体育が「TT・2コマ連続・1-2と合同」の場合に月曜2限に配置
- [ ] → 1-1・1-2の月曜2限・3限 = 4スロット全てに体育（田中+鈴木）が入る
- [ ] トースト「授業を配置しました（2コマ連続・TT・2クラス合同）」と表示される

### 2-6. 上書き確認ダイアログ
- [ ] 既存授業があるセルに別の授業をドロップ → 上書き確認ダイアログが出る
- [ ] キャンセル → 変更なし
- [ ] OK → 上書きされる

## 3. 自動配置（AutoScheduler）

### 3-1. 基本動作
- [ ] 「🤖 自動配置」ボタンをクリック → ダイアログが開く
- [ ] 「全クラス / 空きコマのみ / 分散あり」でプレビュー → 配置予定コマ数が表示される
- [ ] 「実行」→ 時間割が自動描画される
- [ ] Ctrl+Z でまとめてUndoできる

### 3-2. クリア配置
- [ ] 「ゼロから配置」モード → 警告メッセージが出る
- [ ] 実行 → 既存授業がクリアされてから再配置される

### 3-3. 担当教員未設定の科目
- [ ] 担当教員が設定されていない科目がある場合
- [ ] → プレビューの「配置できないコマ」に表示される
- [ ] → 実行しても配置されない（エラーは出ない）

### 3-4. TT・合同・連続コマ設定の反映
- [ ] TT設定の科目が自動配置される → teacherIds に全担当教員が入っている
- [ ] 合同設定の科目が自動配置される → 設定した全クラスに同時配置される
- [ ] 連続コマ設定の科目が自動配置される → N コマ連続で配置される
- [ ] 合同科目は「配置予定コマ数」で重複カウントされない（ユニークな classId-day-period で計算）

### 3-5. 分散配置オプション
- [ ] 「分散あり」で実行 → 同科目が特定曜日に集中しない
- [ ] 「分散なし」で実行 → 早いスロットから順に詰められる

---

## 4. 複数クラス同時配置（教員視点モーダル）

### 4-1. 複数クラス選択
- [ ] 教員の空きセルをクリック → クラス選択チェックボックスが出る
- [ ] 複数クラスにチェック → 全クラスに一括配置される
- [ ] Ctrl+Z で全クラス同時にUndoできる（★ snapshot 修正確認）

### 4-2. 連続コマ反映
- [ ] 2コマ連続設定の科目を複数クラスで選択して配置
- [ ] → 各クラスで後続コマも自動配置される

### 4-3. 学年違い合同確認
- [ ] 1年と2年のクラスを同時選択 → 「学年違いの確認」ダイアログが出る
- [ ] キャンセル → 何も配置されない

---

## 5. Undo/Redo

| 操作 | 期待動作 |
|------|---------|
| 手動配置後に Ctrl+Z | 配置前の状態に戻る |
| 複数クラス登録後に Ctrl+Z | 全変更が一括で戻る |
| 自動配置後に Ctrl+Z | 全自動配置が一括で戻る |
| Ctrl+Z 連打 | 最大20回まで遡れる |
| Ctrl+Y / Ctrl+Shift+Z | Redo できる |

---

## 6. データ保持（LocalStorage）

- [ ] 設定後にページをリロード → クラス別設定が保持されている
- [ ] 合同リンク設定がリロード後も双方向で保持されている
- [ ] 授業の配置がリロード後も保持されている

---

## 7. エッジケース

| ケース | 期待動作 |
|--------|---------|
| 連続コマが最終時限（6限目）にかかる | 範囲内のコマのみ配置し、越えた分は無視 |
| 合同クラスに科目カリキュラムがない | 配置は主クラスのみ。警告なし（blocked に記録） |
| 同一スロットに自動配置が重複 | handledJoint で二重処理防止 |
| LocalStorage 容量超過 | 警告トーストが出る（utils.js 対応済み） |
| 担当教員を削除後に自動配置 | unplaced に記録され、スキップされる |
